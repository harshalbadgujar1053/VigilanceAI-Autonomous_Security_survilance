### VigilanceAI-Autonomous_Security_Survilance

**Problem Statement**

Modern Security Operations Centres are overwhelmed by the sheer volume of alerts generated by SIEM systems, with enterprise environments routinely producing thousands of alerts per day. SOC analysts spend a large share of their working hours investigating alerts that turn out to be false positives, which leads to alert fatigue, delayed response to genuine threats, and a higher mean time to respond. Existing SIEM platforms like Splunk and Elastic can ingest logs and generate rule-based alerts, but they lack the reasoning ability to autonomously investigate an alert, correlate it against external threat intelligence, or produce a human-readable incident report without manual analyst work. Commercial AI-assisted tools such as Microsoft Sentinel Copilot move toward solving this, but they sit behind enterprise licensing, operate as closed systems, don't expose their internal reasoning, and are out of reach for smaller organizations and students. No existing open-source project combines autonomous multi-agent LLM reasoning, RAG over live threat intelligence, and real-time SIEM integration in a single self-hostable pipeline.

**Solution**

ThreatLens is an open-source autonomous SOC co-pilot that closes this gap. It connects a LangChain ReAct agent directly to Wazuh SIEM telemetry, equips the agent with five purpose-built tools (SIEM querying, threat intelligence retrieval, CVE lookup, MITRE ATT&CK mapping, and structured report generation), and grounds its reasoning in a RAG knowledge base built from the MITRE ATT&CK STIX dataset, the NVD CVE feed, and the BRON threat graph. When an alert fires, the agent autonomously decides what to investigate, pulls relevant context, classifies the alert, and produces a structured incident report — replicating the cognitive workflow of a human Tier-1 analyst, with every reasoning step exposed for analyst review. A feedback mechanism lets analysts rate triage quality, building an evaluation dataset that improves the system over time.

**How this differs from existing tools**

Splunk and Elastic SIEM handle log ingestion and dashboards well but stop at rule-based alerting with no reasoning layer, no autonomous investigation, and no threat intel grounding. SOAR platforms like Palo Alto XSOAR and IBM QRadar automate response through static playbooks, but those playbooks don't adapt dynamically the way an LLM agent does, and they don't reason over live threat intel. Commercial AI copilots like Microsoft Sentinel Copilot and Google SecOps add LLM-generated summaries, but they're closed, paid, cloud-only, and don't expose the reasoning chain behind their conclusions. Wazuh, the open-source SIEM ThreatLens builds on, is free and solid at detection but has no AI layer at all. ThreatLens is the only project in this space combining autonomous multi-agent reasoning, live RAG-based threat intelligence, MITRE ATT&CK auto-mapping, and full chain-of-thought transparency — while remaining free, open-source, and self-hostable by any team, not just enterprises with SOAR budgets.

**Tech stack**

SIEM: Wazuh. AI/LLM layer: LangChain ReAct agent running on Ollama/Mistral (local, near-zero cost). Knowledge retrieval: ChromaDB vector store over MITRE ATT&CK STIX, NVD CVE feed, and the BRON threat graph. Backend: FastAPI. Frontend: React. Storage: PostgreSQL for analyst feedback data. Deployment: Docker Compose locally, AWS (free tier) in production. Evaluation dataset: CICIDS-2018.
